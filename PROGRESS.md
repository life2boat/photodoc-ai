🛠 Стандарт: Конвейер загрузки файлов (PhotoDoc AI Core)
Этот протокол обязателен для всех виджетов, предполагающих отправку файлов на сервер.

1. Фронтенд (React / Client Side)
Метод отправки: Использование объекта FormData.

Заголовки (Headers): Категорический запрет на ручную установку Content-Type. Браузер должен автоматически выставлять multipart/form-data с корректным boundary.

Управление состоянием: * Файлы хранятся в массиве selectedFiles.

После успешного ответа сервера (response.ok) выполняется полная очистка состояния: setSelectedFiles([]).

UI принудительно обновляется через updateGallery() или аналогичную функцию для очистки превью.

Безопасность: Блокировка стандартного поведения браузера для Drag-and-Drop на уровне window, чтобы избежать случайного открытия фото в новой вкладке.

2. Бэкенд (FastAPI / Server Side)
Эндпоинт: Единый унифицированный путь /api/order.

Хранение: * Корневая папка: uploads/.

Изоляция: Создание уникальной папки под каждый заказ (Order_YYYYMMDD_HHMMSS или порядковый номер).

Метаданные: Параллельно с сохранением фото сервер обязан генерировать файл order_info.txt со всеми деталями заказа (ФИО, телефон, состав услуг, итоговая стоимость).

3. Администрирование (Admin Panel)
Сборка заказа: Использование io.BytesIO для создания ZIP-архива.

Ресурсосбережение: Архивация происходит исключительно в оперативной памяти «на лету» в момент запроса. Физические ZIP-файлы на диске сервера не создаются.

Почему это важно (Reminder):
Масштабируемость: Мы можем добавить 10 новых типов услуг, не меняя логику обработки файлов.

Стабильность: Этот метод исключает ошибки 422 (неверный формат данных) и 404 (рассинхрон путей).

Порядок: Оператор всегда получает структурированный архив, где фото и описание лежат вместе.
